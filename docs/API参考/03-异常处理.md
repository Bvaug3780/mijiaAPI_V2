# 异常处理

本文档介绍米家API SDK的异常体系、异常类型和处理方法。

## 异常层次结构

所有异常都继承自 `MijiaAPIException` 基类，形成清晰的异常层次：

```
MijiaAPIException (基类)
├── AuthenticationError (认证错误)
│   ├── LoginFailedError (登录失败)
│   └── TokenExpiredError (Token过期)
├── DeviceError (设备错误)
│   ├── DeviceOfflineError (设备离线)
│   ├── DeviceNotFoundError (设备不存在)
│   └── PropertyReadOnlyError (属性只读)
├── NetworkError (网络错误)
│   ├── TimeoutError (超时)
│   └── ConnectionError (连接错误)
└── ValidationError (参数验证错误)
```

## 异常类型详解

### MijiaAPIException

所有异常的基类，包含错误消息、错误码和上下文信息。

**属性：**
- `message: str` - 错误消息
- `code: Optional[int]` - 错误码
- `context: Dict[str, Any]` - 上下文信息

**示例：**
```python
try:
    api.get_devices(home_id)
except MijiaAPIException as e:
    print(f"错误: {e.message}")
    print(f"错误码: {e.code}")
    print(f"上下文: {e.context}")
```

### AuthenticationError

认证相关的错误，包括登录失败、Token过期等。

**子类：**
- `LoginFailedError` - 登录失败（用户名密码错误、验证码错误等）
- `TokenExpiredError` - Token过期（需要重新登录）

**示例：**
```python
from mijiaAPI_V2.domain.exceptions import TokenExpiredError

try:
    devices = api.get_devices(home_id)
except TokenExpiredError:
    print("Token已过期，请重新登录")
    # 重新登录逻辑
    api = create_api_client(username, password)
```

### DeviceError

设备相关的错误，包括设备离线、设备不存在、属性只读等。

**子类：**
- `DeviceOfflineError` - 设备离线（无法控制离线设备）
- `DeviceNotFoundError` - 设备不存在（设备ID错误或已删除）
- `PropertyReadOnlyError` - 属性只读（尝试写入只读属性）

**示例：**
```python
from mijiaAPI_V2.domain.exceptions import DeviceOfflineError, PropertyReadOnlyError

try:
    api.set_device_property(device_id, siid=2, piid=1, value=True)
except DeviceOfflineError:
    print("设备离线，无法控制")
except PropertyReadOnlyError:
    print("该属性为只读，无法修改")
```

### NetworkError

网络相关的错误，包括超时、连接失败等。

**子类：**
- `TimeoutError` - 请求超时
- `ConnectionError` - 连接失败（网络不可达、DNS解析失败等）

**示例：**
```python
from mijiaAPI_V2.domain.exceptions import TimeoutError, ConnectionError

try:
    devices = api.get_devices(home_id)
except TimeoutError:
    print("请求超时，请检查网络连接")
except ConnectionError:
    print("网络连接失败")
```

### ValidationError

参数验证错误，当传入的参数不符合要求时抛出。

**示例：**
```python
from mijiaAPI_V2.domain.exceptions import ValidationError

try:
    api.set_device_property(device_id, siid=2, piid=1, value="invalid")
except ValidationError as e:
    print(f"参数验证失败: {e.message}")
```

## 异常处理最佳实践

### 1. 捕获具体异常

优先捕获具体的异常类型，而不是通用的 `Exception`：

```python
# ✓ 推荐
try:
    api.set_device_property(device_id, siid=2, piid=1, value=True)
except DeviceOfflineError:
    print("设备离线")
except PropertyReadOnlyError:
    print("属性只读")
except DeviceError as e:
    print(f"设备错误: {e}")

# ✗ 不推荐
try:
    api.set_device_property(device_id, siid=2, piid=1, value=True)
except Exception as e:
    print(f"错误: {e}")
```

### 2. 使用异常层次

利用异常层次结构，可以在不同粒度上捕获异常：

```python
# 捕获所有认证相关错误
try:
    api.get_devices(home_id)
except AuthenticationError:
    print("认证失败，请重新登录")

# 捕获特定的Token过期错误
try:
    api.get_devices(home_id)
except TokenExpiredError:
    print("Token过期")
except LoginFailedError:
    print("登录失败")
```

### 3. 记录异常上下文

使用异常的 `context` 属性记录额外信息：

```python
try:
    api.set_device_property(device_id, siid=2, piid=1, value=True)
except MijiaAPIException as e:
    print(f"错误: {e.message}")
    if e.context:
        print(f"设备ID: {e.context.get('device_id')}")
        print(f"属性: siid={e.context.get('siid')}, piid={e.context.get('piid')}")
```

### 4. 优雅降级

对于非关键操作，可以捕获异常并继续执行：

```python
devices = api.get_devices(home_id)

for device in devices:
    try:
        # 尝试获取设备状态
        properties = api.get_device_properties(device.did)
        print(f"{device.name}: {properties}")
    except DeviceOfflineError:
        # 设备离线，跳过
        print(f"{device.name}: 离线")
        continue
    except DeviceError as e:
        # 其他设备错误，记录但继续
        print(f"{device.name}: 错误 - {e.message}")
        continue
```

### 5. 重试机制

对于临时性错误（如网络超时），可以实现重试机制：

```python
import time
from mijiaAPI_V2.domain.exceptions import TimeoutError, NetworkError

def get_devices_with_retry(api, home_id, max_retries=3):
    """带重试的获取设备列表"""
    for attempt in range(max_retries):
        try:
            return api.get_devices(home_id)
        except TimeoutError:
            if attempt < max_retries - 1:
                print(f"超时，重试 {attempt + 1}/{max_retries}...")
                time.sleep(2 ** attempt)  # 指数退避
            else:
                raise
        except NetworkError:
            if attempt < max_retries - 1:
                print(f"网络错误，重试 {attempt + 1}/{max_retries}...")
                time.sleep(2 ** attempt)
            else:
                raise
```

### 6. 自定义异常处理器

创建统一的异常处理器：

```python
from mijiaAPI_V2.domain.exceptions import (
    MijiaAPIException,
    AuthenticationError,
    DeviceError,
    NetworkError,
)

def handle_api_exception(e: MijiaAPIException) -> None:
    """统一的异常处理器"""
    if isinstance(e, AuthenticationError):
        print(f"认证错误: {e.message}")
        # 触发重新登录流程
    elif isinstance(e, DeviceError):
        print(f"设备错误: {e.message}")
        # 记录设备错误日志
    elif isinstance(e, NetworkError):
        print(f"网络错误: {e.message}")
        # 检查网络连接
    else:
        print(f"未知错误: {e.message}")

# 使用
try:
    api.get_devices(home_id)
except MijiaAPIException as e:
    handle_api_exception(e)
```

## 错误码映射

SDK 会自动将 API 返回的错误码映射到对应的异常类型：

| 错误码 | 异常类型 | 说明 |
|--------|----------|------|
| 401 | TokenExpiredError | Token过期 |
| 404 | DeviceNotFoundError | 设备不存在 |
| 408 | TimeoutError | 请求超时 |
| 500 | MijiaAPIException | 服务器错误 |

**示例：**
```python
from mijiaAPI_V2.domain.exceptions import get_exception_by_code

# API返回错误码
error_code = 401
error_message = "Token已过期"

# 获取对应的异常
exception = get_exception_by_code(error_code, error_message)
raise exception  # 抛出 TokenExpiredError
```

## 完整示例

```python
from mijiaAPI_V2 import create_api_client_from_file
from mijiaAPI_V2.domain.exceptions import (
    MijiaAPIException,
    AuthenticationError,
    TokenExpiredError,
    DeviceError,
    DeviceOfflineError,
    NetworkError,
    TimeoutError,
)

def main():
    try:
        # 创建API客户端
        api = create_api_client_from_file()
        
        # 获取家庭列表
        homes = api.get_homes()
        home = homes[0]
        
        # 获取设备列表
        devices = api.get_devices(home.id)
        
        # 控制设备
        for device in devices:
            try:
                if device.is_online():
                    api.set_device_property(
                        device.did,
                        siid=2,
                        piid=1,
                        value=True
                    )
                    print(f"✓ {device.name}: 控制成功")
            except DeviceOfflineError:
                print(f"✗ {device.name}: 设备离线")
            except DeviceError as e:
                print(f"✗ {device.name}: {e.message}")
                
    except TokenExpiredError:
        print("Token已过期，请重新登录")
    except AuthenticationError as e:
        print(f"认证失败: {e.message}")
    except TimeoutError:
        print("请求超时，请检查网络连接")
    except NetworkError as e:
        print(f"网络错误: {e.message}")
    except MijiaAPIException as e:
        print(f"API错误: {e.message}")
        if e.code:
            print(f"错误码: {e.code}")

if __name__ == "__main__":
    main()
```
