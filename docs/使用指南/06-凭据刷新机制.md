# 凭据刷新机制

## 概述

米家API的凭据（Credential）具有有效期限制，默认约为7天。为了避免频繁重新登录，系统提供了凭据刷新功能，允许在凭据有效期内刷新以延长使用时间。

## 工作原理

### 1. 凭据结构

完整的凭据对象包含以下关键字段：

```python
{
    "user_id": "用户ID",
    "service_token": "服务Token，用于API调用",
    "ssecurity": "安全密钥，用于请求签名",
    "pass_token": "通行Token，用于刷新凭据",  # 关键字段
    "c_user_id": "Cookie用户ID",
    "device_id": "设备ID",
    "user_agent": "User-Agent",
    "expires_at": "过期时间"
}
```

其中 `pass_token` 是刷新凭据的关键，只有包含此字段的凭据才能被刷新。

### 2. 刷新流程

刷新凭据的过程如下：

1. 携带现有的 `passToken` 访问小米账号的 `serviceLogin` 接口
2. 如果 `passToken` 仍然有效，服务器返回新的 `location` URL
3. 访问 `location` URL 获取新的 `serviceToken` 和 `ssecurity`
4. 构建新的凭据对象，保留原有的 `passToken`
5. 更新过期时间

### 3. 无限刷新

理论上，只要在凭据过期前进行刷新，就可以获得新的有效期。通过定期刷新（例如每天刷新一次），可以保持凭据永久有效，无需重新登录。

## 使用方法

### 基本用法

```python
from mijiaAPI_V2 import create_auth_service

# 创建认证服务
auth_service = create_auth_service()

# 加载现有凭据
credential = auth_service.load_credential()

# 检查凭据状态
if credential.is_expired():
    print("凭据已过期")
elif credential.expires_in() < 86400:  # 少于1天
    print("凭据即将过期，建议刷新")

# 刷新凭据
try:
    new_credential = auth_service.refresh_credential(credential)
    auth_service.save_credential(new_credential)
    print(f"刷新成功，新的过期时间: {new_credential.expires_at}")
except Exception as e:
    print(f"刷新失败: {e}")
```

### 自动刷新策略

建议实现自动刷新机制，在凭据即将过期时自动刷新：

```python
from datetime import datetime, timedelta
from mijiaAPI_V2 import create_auth_service, create_api_client_from_file

def get_api_with_auto_refresh():
    """获取API客户端，自动刷新凭据"""
    auth_service = create_auth_service()
    credential = auth_service.load_credential()
    
    if not credential:
        raise Exception("凭据不存在，请先登录")
    
    # 如果凭据在1天内过期，自动刷新
    if credential.expires_in() < 86400:
        print("凭据即将过期，正在自动刷新...")
        try:
            new_credential = auth_service.refresh_credential(credential)
            auth_service.save_credential(new_credential)
            print("凭据刷新成功")
        except Exception as e:
            print(f"凭据刷新失败: {e}")
            print("请重新登录")
            raise
    
    return create_api_client_from_file()

# 使用
api = get_api_with_auto_refresh()
homes = api.get_homes()
```

### 定时刷新任务

对于长期运行的应用，可以设置定时任务定期刷新凭据：

```python
import schedule
import time
from mijiaAPI_V2 import create_auth_service

def refresh_credential_job():
    """定时刷新凭据的任务"""
    try:
        auth_service = create_auth_service()
        credential = auth_service.load_credential()
        
        if credential and not credential.is_expired():
            new_credential = auth_service.refresh_credential(credential)
            auth_service.save_credential(new_credential)
            print(f"凭据刷新成功，新的过期时间: {new_credential.expires_at}")
        else:
            print("凭据已过期或不存在，需要重新登录")
    except Exception as e:
        print(f"凭据刷新失败: {e}")

# 每天凌晨3点刷新凭据
schedule.every().day.at("03:00").do(refresh_credential_job)

# 运行调度器
while True:
    schedule.run_pending()
    time.sleep(60)
```

## 注意事项

### 1. passToken 的重要性

- 只有包含 `pass_token` 字段的凭据才能被刷新
- 旧版本的凭据可能不包含此字段，需要重新登录
- `pass_token` 在刷新过程中保持不变，不会更新

### 2. 刷新时机

- 建议在凭据过期前1-2天进行刷新
- 不要等到凭据完全过期后再刷新，可能会失败
- 刷新频率不宜过高，建议每天刷新一次即可

### 3. 刷新失败处理

刷新可能失败的原因：

- 凭据缺少 `pass_token` 字段
- `pass_token` 已失效（通常在凭据过期后）
- 网络连接问题
- 小米服务器拒绝刷新请求

失败后的处理：

```python
try:
    new_credential = auth_service.refresh_credential(credential)
    auth_service.save_credential(new_credential)
except TokenExpiredError as e:
    if "缺少passToken" in str(e):
        print("凭据格式过旧，请重新登录")
    else:
        print("凭据已失效，请重新登录")
    # 执行重新登录流程
    new_credential = auth_service.login_by_qrcode()
    auth_service.save_credential(new_credential)
```

### 4. 兼容性

- 新版本凭据包含 `pass_token` 字段，支持刷新
- 旧版本凭据不包含此字段，加载时会自动设置为空字符串
- 尝试刷新旧版本凭据会提示需要重新登录

## 最佳实践

1. **首次登录后保存凭据**
   ```python
   credential = auth_service.login_by_qrcode()
   auth_service.save_credential(credential)
   ```

2. **使用前检查凭据状态**
   ```python
   if credential.is_expired() or credential.expires_in() < 86400:
       # 刷新或重新登录
   ```

3. **实现自动刷新机制**
   - 在应用启动时检查并刷新
   - 设置定时任务定期刷新
   - 在API调用失败时尝试刷新

4. **妥善保管凭据文件**
   - 凭据文件包含敏感信息，不要提交到版本控制
   - 设置适当的文件权限（默认为 0o600）
   - 定期备份凭据文件
