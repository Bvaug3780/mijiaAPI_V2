# 开发环境搭建

## 系统要求

- **操作系统**：macOS、Linux、Windows
- **Python版本**：3.9+
- **包管理器**：uv（推荐）或 pip

## 安装 uv

### macOS / Linux

```bash
# 使用官方安装脚本
curl -LsSf https://astral.sh/uv/install.sh | sh

# 或使用 Homebrew
brew install uv
```

### Windows

```powershell
# 使用 PowerShell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
```

### 验证安装

```bash
uv --version
```

## 克隆项目

```bash
# 克隆仓库
git clone https://github.com/your-org/mijia-api.git
cd mijia-api

# 查看项目结构
ls -la
```

## 安装依赖

### 基础依赖

```bash
# 创建虚拟环境并安装依赖
uv sync
```

这会自动：
1. 创建 `.venv` 虚拟环境
2. 安装 `pyproject.toml` 中定义的所有依赖
3. 安装项目本身（可编辑模式）

### 开发依赖

```bash
# 安装开发工具（pytest、mypy、flake8等）
uv sync --extra dev
```

### 可选依赖

```bash
# Redis支持
uv sync --extra redis

# CLI工具
uv sync --extra cli

# 文档生成
uv sync --extra docs

# 安装所有依赖
uv sync --all-extras
```

## 激活虚拟环境

### 方式一：使用 uv run（推荐）

```bash
# 直接运行命令，无需激活
uv run python script.py
uv run pytest
uv run mypy mijiaAPI_V2/
```

### 方式二：手动激活

```bash
# macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate

# 验证
which python  # 应该指向 .venv/bin/python
```

## IDE配置

### VS Code

创建 `.vscode/settings.json`：

```json
{
  "python.defaultInterpreterPath": "${workspaceFolder}/.venv/bin/python",
  "python.linting.enabled": true,
  "python.linting.flake8Enabled": true,
  "python.linting.mypyEnabled": true,
  "python.linting.pylintEnabled": true,
  "python.formatting.provider": "black",
  "python.testing.pytestEnabled": true,
  "python.testing.pytestArgs": ["tests"],
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.organizeImports": true
  }
}
```

### PyCharm

1. 打开项目
2. File → Settings → Project → Python Interpreter
3. 选择 `.venv/bin/python`
4. 启用 pytest 作为测试运行器

## 验证安装

### 运行测试

```bash
# 运行所有测试
uv run pytest

# 运行单元测试
uv run pytest tests/unit/

# 运行集成测试
uv run pytest tests/integration/

# 生成覆盖率报告
uv run pytest --cov=mijiaAPI_V2 --cov-report=html
```

### 代码质量检查

```bash
# 类型检查
uv run mypy mijiaAPI_V2/

# 代码风格检查
uv run flake8 mijiaAPI_V2/

# 代码质量检查
uv run pylint mijiaAPI_V2/
```

### 代码格式化

```bash
# 格式化代码
uv run black mijiaAPI_V2/

# 排序导入
uv run isort mijiaAPI_V2/
```

## 使用 Makefile（推荐）

项目提供了 Makefile 简化常用命令：

```bash
# 查看所有可用命令
make help

# 安装依赖
make install

# 运行测试
make test

# 测试覆盖率
make test-cov

# 代码格式化
make format

# 代码检查
make lint

# 类型检查
make type-check

# 清理缓存
make clean
```

## 开发工作流

### 1. 创建功能分支

```bash
git checkout -b feature/your-feature-name
```

### 2. 编写代码

遵循项目规范：
- 使用类型注解
- 编写中文文档字符串
- 添加单元测试

### 3. 运行测试

```bash
# 运行相关测试
uv run pytest tests/unit/test_your_module.py

# 确保所有测试通过
make test
```

### 4. 代码检查

```bash
# 格式化代码
make format

# 运行所有检查
make lint
make type-check
```

### 5. 提交代码

```bash
git add .
git commit -m "feat: 添加新功能"
git push origin feature/your-feature-name
```

## 常见问题

### Q1: uv sync 失败

```bash
# 清理缓存后重试
rm -rf .venv
uv sync
```

### Q2: 导入错误

```bash
# 确保虚拟环境已激活
source .venv/bin/activate

# 或使用 uv run
uv run python your_script.py
```

### Q3: 测试失败

```bash
# 清理缓存
make clean

# 重新运行测试
make test
```

### Q4: mypy 报错

```bash
# 安装类型存根
uv pip install types-requests types-redis

# 或忽略特定错误
# type: ignore
```
