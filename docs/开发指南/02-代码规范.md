## Python 代码规范

### 命名约定（严格遵循 PEP 8）

#### 函数和变量：snake_case

```python
# ✅ 好
def get_device_list(home_id: str) -> List[Device]:
    device_count = len(devices)
    is_online = device.status == 1

# ❌ 不好
def GetDeviceList(homeId: str) -> List[Device]:
    deviceCount = len(devices)
    IsOnline = device.status == 1
```

#### 类：PascalCase

```python
# ✅ 好
class DeviceService:
    pass

class HttpClient:
    pass

# ❌ 不好
class device_service:
    pass

class http_client:
    pass
```

#### 常量：UPPER_SNAKE_CASE

```python
# ✅ 好
MAX_RETRY_COUNT = 3
DEFAULT_TIMEOUT = 30
API_BASE_URL = "https://api.mina.mi.com"

# ❌ 不好
maxRetryCount = 3
default_timeout = 30
apiBaseUrl = "https://api.mina.mi.com"
```

#### 私有成员：前缀 _

```python
# ✅ 好
class DeviceService:
    def __init__(self):
        self._device_repo = DeviceRepositoryImpl()
        self._cache = CacheManager()
    
    def _validate_device(self, device: Device) -> bool:
        """私有方法"""
        pass

# ❌ 不好
class DeviceService:
    def __init__(self):
        self.device_repo = DeviceRepositoryImpl()
        self.cache = CacheManager()
    
    def validate_device(self, device: Device) -> bool:
        """应该是私有方法"""
        pass
```

#### 禁止使用拼音命名

```python
# ❌ 不好
def huoqu_shebei(jiating_id: str):
    shebei_liebiao = []

# ✅ 好
def get_devices(home_id: str):
    device_list = []
```

## 代码注释

### 注释原则

1. **说明"为什么"而不是"是什么"**
2. **复杂逻辑必须添加注释**
3. **避免冗余注释**

### 好的注释

```python
# ✅ 好：解释为什么这样做
def get_devices(self, home_id: str, credential: Credential) -> List[Device]:
    # 先检查缓存，避免频繁调用API
    cached = self._cache.get(f"devices:{home_id}")
    if cached:
        return cached
    
    # 从API获取设备列表
    devices = self._device_repo.get_by_home(home_id, credential)
    
    # 缓存5分钟，因为设备列表变化不频繁
    self._cache.set(f"devices:{home_id}", devices, ttl=300)
    
    return devices
```

### 不好的注释

```python
# ❌ 不好：注释重复了代码
def get_devices(self, home_id: str, credential: Credential) -> List[Device]:
    # 获取缓存
    cached = self._cache.get(f"devices:{home_id}")
    # 如果缓存存在
    if cached:
        # 返回缓存
        return cached
    
    # 获取设备
    devices = self._device_repo.get_by_home(home_id, credential)
    
    # 设置缓存
    self._cache.set(f"devices:{home_id}", devices, ttl=300)
    
    # 返回设备
    return devices
```

## 错误处理

### 使用具体异常

```python
# ✅ 好：捕获具体异常
try:
    device = api.get_device(device_id)
except DeviceNotFoundError:
    logger.warning(f"设备不存在: {device_id}")
except TokenExpiredError:
    logger.error("凭据已过期")
    refresh_credential()
except NetworkError as e:
    logger.error(f"网络错误: {e}")

# ❌ 不好：捕获所有异常
try:
    device = api.get_device(device_id)
except Exception as e:
    logger.error(f"错误: {e}")
```

### 异常链

```python
# ✅ 好：保留异常链
try:
    response = requests.get(url)
    response.raise_for_status()
except requests.HTTPError as e:
    raise NetworkError(f"HTTP请求失败: {url}") from e

# ❌ 不好：丢失原始异常
try:
    response = requests.get(url)
    response.raise_for_status()
except requests.HTTPError:
    raise NetworkError(f"HTTP请求失败: {url}")
```

### 资源清理

```python
# ✅ 好：使用上下文管理器
with open("file.txt", "r") as f:
    data = f.read()

# ✅ 好：使用try-finally
file = open("file.txt", "r")
try:
    data = file.read()
finally:
    file.close()

# ❌ 不好：可能泄漏资源
file = open("file.txt", "r")
data = file.read()
file.close()  # 如果read()抛出异常，不会执行
```

## 函数设计

### 单一职责

```python
# ✅ 好：每个函数只做一件事
def get_devices(home_id: str) -> List[Device]:
    """获取设备列表"""
    return self._device_repo.get_by_home(home_id)

def filter_online_devices(devices: List[Device]) -> List[Device]:
    """筛选在线设备"""
    return [d for d in devices if d.status == 1]

def sort_devices_by_name(devices: List[Device]) -> List[Device]:
    """按名称排序设备"""
    return sorted(devices, key=lambda d: d.name)

# ❌ 不好：一个函数做太多事
def get_online_devices_sorted(home_id: str) -> List[Device]:
    """获取在线设备并排序"""
    devices = self._device_repo.get_by_home(home_id)
    online = [d for d in devices if d.status == 1]
    return sorted(online, key=lambda d: d.name)
```

## Git Commit 规范

### Commit 消息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型

- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式（不影响代码运行）
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

### 示例

```bash
# 新功能
git commit -m "feat(device): 添加批量控制设备功能"

# 修复bug
git commit -m "fix(auth): 修复凭据过期时的刷新逻辑"

# 文档更新
git commit -m "docs(api): 更新API客户端文档"

# 重构
git commit -m "refactor(cache): 重构缓存管理器，支持Redis"

# 详细消息
git commit -m "feat(device): 添加设备状态监听功能

- 实现WebSocket连接
- 添加状态变化回调
- 支持断线重连

Closes #123"
```

## 性能优化

### 使用生成器

```python
# ✅ 好：使用生成器，节省内存
def get_all_devices() -> Generator[Device, None, None]:
    for home in homes:
        for device in get_devices(home.id):
            yield device

# ❌ 不好：一次性加载所有数据
def get_all_devices() -> List[Device]:
    all_devices = []
    for home in homes:
        all_devices.extend(get_devices(home.id))
    return all_devices
```

### 避免重复计算

```python
# ✅ 好：缓存计算结果
@lru_cache(maxsize=128)
def calculate_statistics(devices: Tuple[Device, ...]) -> Dict:
    return {
        "total": len(devices),
        "online": sum(1 for d in devices if d.status == 1),
    }

# ❌ 不好：每次都重新计算
def calculate_statistics(devices: List[Device]) -> Dict:
    return {
        "total": len(devices),
        "online": sum(1 for d in devices if d.status == 1),
    }
```
