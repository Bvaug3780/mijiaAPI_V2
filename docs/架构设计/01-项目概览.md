# 项目概览

## 项目简介

米家API SDK（mijiaAPI_V2）是一个现代化的Python SDK，用于与小米智能家居设备进行交互。本项目采用领域驱动设计（DDD）和分层架构，提供了清晰、可维护、可扩展的代码结构。

## 核心特性

### 1. 凭据与操作分离
- **设计理念**：将用户认证凭据（Credential）作为独立的领域对象，与API操作完全解耦
- **优势**：
  - 支持多用户并发场景
  - 凭据可以独立管理、存储、刷新
  - 便于实现凭据的加密存储和安全传输

### 2. 清晰的分层架构
项目采用经典的分层架构模式：

```
┌─────────────────────────────────────┐
│      API客户端层 (api_client.py)    │  ← 对外统一接口
├─────────────────────────────────────┤
│      服务层 (services/)              │  ← 业务逻辑封装
├─────────────────────────────────────┤
│      仓储层 (repositories/)          │  ← 数据访问抽象
├─────────────────────────────────────┤
│      基础设施层 (infrastructure/)    │  ← 技术实现
├─────────────────────────────────────┤
│      领域层 (domain/)                │  ← 核心业务实体
└─────────────────────────────────────┘
```

**为什么采用分层架构？**
- **关注点分离**：每一层只关注自己的职责
- **依赖倒置**：高层不依赖低层，都依赖抽象
- **易于测试**：每一层可以独立测试
- **易于替换**：可以轻松替换某一层的实现

### 3. 多用户并发支持
- **实现方式**：每个用户拥有独立的API客户端实例
- **缓存隔离**：使用用户ID作为缓存命名空间
- **状态隔离**：不同用户的操作互不影响

### 4. 完整的类型注解
- **类型安全**：所有公共API都有完整的类型提示
- **IDE支持**：提供智能代码补全和类型检查
- **静态检查**：支持mypy进行静态类型检查

### 5. 智能缓存管理
三层缓存架构：
- **内存缓存**：最快，适合频繁访问的数据
- **Redis缓存**：分布式，适合多进程/多服务器场景
- **文件缓存**：持久化，适合长期存储

### 6. 异步API支持
- **同步API**：`MijiaAPI` - 简单直接，适合脚本和简单应用
- **异步API**：`AsyncMijiaAPI` - 高性能，适合Web应用和并发场景

## 设计原则

### 1. SOLID原则

#### 单一职责原则 (SRP)
每个类只负责一件事：
- `HttpClient` 只负责HTTP通信
- `CacheManager` 只负责缓存管理
- `DeviceService` 只负责设备相关业务逻辑

#### 开闭原则 (OCP)
对扩展开放，对修改关闭：
- 通过接口定义仓储抽象
- 可以轻松添加新的仓储实现（如数据库仓储）

#### 里氏替换原则 (LSP)
子类可以替换父类：
- 所有仓储实现都遵循相同的接口
- 可以无缝切换不同的实现

#### 接口隔离原则 (ISP)
接口应该小而专注：
- `IDeviceRepository` 只定义设备相关操作
- `IHomeRepository` 只定义家庭相关操作

#### 依赖倒置原则 (DIP)
依赖抽象而非具体实现：
- 服务层依赖仓储接口，而非具体实现
- 通过依赖注入组装组件

### 2. 领域驱动设计 (DDD)

#### 领域模型
核心业务实体：
- `Credential` - 用户凭据
- `Device` - 设备
- `Home` - 家庭
- `Scene` - 智能

#### 仓储模式
数据访问抽象：
- 隐藏数据访问细节
- 提供领域对象的持久化

#### 服务层
业务逻辑封装：
- 协调多个仓储
- 实现复杂业务规则

## 快速开始

### 基本使用流程

```python
from mijiaAPI_V2 import create_api_client, create_auth_service

# 1. 获取凭据
auth_service = create_auth_service()
credential = auth_service.login_by_qrcode()

# 2. 创建API客户端
api = create_api_client(credential)

# 3. 获取家庭列表
homes = api.get_homes()

# 4. 获取设备列表
devices = api.get_devices(home_id=homes[0].id)

# 5. 控制设备
api.control_device(device_id="xxx", siid=2, piid=1, value=True)
```
